테스트 흐름 목록

1. 로그인 성공 시나리오:
    * 로그인 페이지로 이동.
    * 유효한 이메일과 비밀번호 입력.
    * 캡챠 위젯이 로드되고 (모의) 토큰이 생성되는 것을 확인.
    * 로그인 버튼 클릭.
    * "로그인 성공!" 메시지가 표시되는 것을 확인.
    * REDIRECT_DELAY 시간 후 메인 페이지(PATHS.ROOT)로 성공적으로 리다이렉트되는 것을 확인.


2. 로그인 실패 시나리오 (잘못된 자격 증명):
    * 로그인 페이지로 이동.
    * 잘못된 이메일 또는 비밀번호 입력.
    * 로그인 버튼 클릭.
    * "이메일/비밀번호가 올바르지 않습니다."와 같은 에러 토스트 메시지가 표시되는 것을 확인.
    * 폼 필드에 에러 스타일이 적용되는지 확인 (선택 사항).


3. 회원가입 성공 시나리오:
    * 회원가입 페이지로 이동.
    * 유효한 이메일, 비밀번호, 비밀번호 확인 입력.
    * 캡챠 위젯 로드 및 토큰 생성 확인.
    * 회원가입 버튼 클릭.
    * "가입 성공" 메시지 또는 이메일 확인 요청 메시지 등 성공 피드백 확인.
    * 필요시 로그인 페이지로 리다이렉트되는 것을 확인.


4. 회원가입 실패 시나리오 (이미 등록된 이메일):
    * 회원가입 페이지로 이동.
    * 이미 등록된 이메일 입력.
    * 회원가입 버튼 클릭.
    * "이미 등록된 이메일 주소입니다."와 같은 에러 토스트 메시지가 표시되는 것을 확인.


5. 비밀번호 찾기 시나리오:
    * 비밀번호 찾기 페이지로 이동.
    * 가입된 이메일 주소 입력.
    * 캡챠 위젯 로드 및 토큰 생성 확인.
    * "비밀번호 찾기" 버튼 클릭.
    * "메일을 확인해주세요."와 같은 성공 메시지 확인.


6. 비밀번호 재설정 시나리오 (이메일 링크를 통한 접근):
    * (이메일 링크를 시뮬레이션하여) 비밀번호 재설정 페이지로 직접 이동.
    * 새로운 비밀번호 입력.
    * "비밀번호 수정" 버튼 클릭.
    * "수정되었습니다!"와 같은 성공 메시지 확인.
    * 로그인 페이지로 리다이렉트되는 것을 확인.


7. 방문자 접속 시나리오:
    * 로그인 페이지로 이동.
    * "방문자로 접속하기" 버튼 클릭.
    * 익명 로그인 성공 후 메인 페이지로 리다이렉트되는 것을 확인.


8. 로그아웃 시나리오:
    * 로그인 후 메인 페이지로 이동.
    * 로그아웃 버튼 클릭 (메인 페이지에 로그아웃 기능이 있다고 가정).
    * 로그인 페이지로 성공적으로 리다이렉트되는 것을 확인.
    * 로그인 페이지에서 캡챠 위젯이 다시 로드되는 것을 확인 (캡챠 토큰 초기화 검증).
