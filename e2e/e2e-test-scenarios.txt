# E2E 테스트 시나리오 목록

## 1. 주요 기능 테스트

### 1-1. 메뉴 관리 (Menu)

- 시나리오 1-1-1: 신규 메뉴 카테고리 및 메뉴 항목 추가
  - 단계:
    1. 하단 탭에서 '메뉴'를 선택합니다.
    2. 우측 하단의 위젯을 열고 '분류 추가'를 클릭합니다.
    3. 모달에서 새로운 분류명(예: '음료')을 입력하고 '추가하기'를 누릅니다.
    4. 생성된 '음료' 카테고리에서 '상품 추가' 버튼을 클릭합니다.
    5. 메뉴 추가 모달에서 사진, 음식명, 가격, 태그 등 모든 필드를 채우고 '추가하기'를 누릅니다.

  - 예상 결과:
    - '음료' 카테고리가 메뉴 탭에 나타나야 합니다.
    - 새로 추가한 메뉴가 '음료' 카테고리 하위에 정상적으로 표시되어야 합니다.

- 시나리오 1-1-2: 메뉴 항목 수정 및 삭제
  - 단계:
    1. '메뉴' 탭에서 수정할 메뉴 항목을 클릭합니다.
    2. 메뉴 수정 모달에서 가격을 변경하고 '수정하기'를 누릅니다.
    3. 다시 동일한 메뉴 항목을 클릭합니다.
    4. '삭제하기' 버튼을 누릅니다.

  - 예상 결과:
    - 메뉴의 가격 정보가 화면에 올바르게 반영되어야 합니다.
    - 메뉴 항목이 목록에서 완전히 사라져야 합니다.

- 시나리오 1-1-3: 메뉴 카테고리 수정 및 삭제
  - 단계:
    1. 위젯에서 '분류 수정'을 클릭합니다.
    2. 수정할 카테고리를 선택하고 새 이름을 입력한 뒤 '수정하기'를 누릅니다.
    3. 위젯에서 '분류 삭제'를 클릭합니다.
    4. 삭제할 카테고리를 선택하고 '삭제하기'를 누릅니다.

  - 예상 결과:
    - 카테고리 이름이 성공적으로 변경되어야 합니다.
    - 해당 카테고리와 하위 메뉴들이 모두 화면에서 사라져야 합니다.

### 1-2. 좌석 관리 (Table)

- 시나리오 1-2-1: 신규 좌석 생성
  - 단계:
    1. 하단 탭에서 '좌석'을 선택합니다.
    2. 위젯을 열고 '좌석 추가'를 클릭합니다.
    3. 새로운 좌석이 캔버스 중앙에 나타나는지 확인합니다.
    4. '추가하기' 버튼을 눌러 저장합니다.

  - 예상 결과:
    - 새로운 좌석이 생성되고, 페이지를 새로고침해도 위치가 유지되어야 합니다.

- 시나리오 1-2-2: 좌석 위치 및 크기 변경
  - 단계:
    1. 위젯에서 '좌석 수정'을 클릭하여 편집 모드로 전환합니다.
    2. 수정할 좌석을 선택합니다.
    3. 좌석을 드래그하여 다른 위치로 이동시킵니다.
    4. 좌석의 모서리를 드래그하여 크기를 조절합니다.
    5. '수정하기' 버튼을 눌러 저장합니다.

  - 예상 결과:
    - 좌석의 위치와 크기가 변경된 상태로 저장되어야 합니다.

- 시나리오 1-2-3: 좌석 QR코드 확인 및 삭제
  - 단계:
    1. 편집 모드가 아닌 상태에서 특정 좌석을 클릭합니다.
    2. 좌석 정보 모달이 나타나는지 확인합니다.
    3. 모달 내에서 QR 코드가 정상적으로 보이는지 확인합니다.
    4. 위젯에서 '좌석 삭제'를 클릭하고 해당 좌석을 선택한 뒤 '삭제하기'를 누릅니다.

  - 예상 결과:
    - 좌석 정보 모달과 QR 코드가 올바르게 표시되어야 합니다.
    - 좌석이 캔버스에서 완전히 삭제되어야 합니다.

### 1-3. 주문 처리 (Order)

- 시나리오 1-3-1: 신규 주문 접수 및 완료 처리
  - 단계:
    1. (사전 준비) 테스트 DB에 새로운 주문을 생성합니다.
    2. 하단 탭에서 '주문'을 선택합니다.
    3. '접수' 탭에 새로운 주문 카드가 나타나는지 확인합니다.
    4. 주문 카드에 표시된 테이블 번호, 메뉴명, 수량, 경과 시간이 올바른지 확인합니다.
    5. 해당 주문 카드의 '조리 완료' 버튼을 클릭합니다.
    6. 상단의 '완료' 탭을 클릭합니다.

  - 예상 결과:
    - '접수' 탭에서 주문 카드가 사라져야 합니다.
    - '완료' 탭으로 이동했을 때, 해당 주문 카드가 비활성화된 상태로 표시되어야 합니다.

### 1-4. 실시간 요청 처리 (Request)

- 시나리오 1-4-1: 고객 요청 알림 확인 및 처리
  - 단계:
    1. (사전 준비) 테스트 DB에 새로운 고객 요청(예: 물, 수저)을 생성합니다.
    2. 화면 좌측 하단에 요청 알림이 나타나는지 확인합니다.
    3. 알림에 표시된 테이블 번호와 요청 내용이 올바른지 확인합니다.
    4. 알림 메시지의 'x' 버튼을 클릭하여 닫습니다.

  - 예상 결과:
    - 실시간 요청 알림이 즉시 나타나야 합니다.
    - 'x' 버튼 클릭 시 알림이 사라져야 합니다.

## 2. 추가 테스트 (예외, UI/UX, 실시간)

### 2-1. 예외 및 경계 값 테스트 (Exception & Edge Case)

- 시나리오 2-1-1: 데이터 미존재
  - 시나리오: 메뉴, 좌석, 주문 데이터가 하나도 없는 초기 상태에서 각 탭(메뉴, 좌석, 주문)에 진입합니다.

  - 예상 결과: "표시할 데이터가 없습니다" 또는 "위젯에서 데이터를 생성해주세요"와 같은 안내 문구가 각 화면에 올바르게 표시되어야 합니다.

- 시나리오 2-1-2: 입력 값 유효성 검사
  - 시나리오:
    - 메뉴 카테고리 추가 시, 이름 없이 '추가하기'를 시도합니다.
    - 메뉴 추가 시, 음식명이나 가격을 입력하지 않고 '추가하기'를 시도합니다.
    - 가격을 음수로 입력합니다.

  - 예상 결과: 각 시나리오에 맞는 유효성 검사 에러 메시지(예: "분류를 입력해주세요", "가격을 입력해주세요")가 alert 창으로 표시되어야 합니다.

- 시나리오 2-1-3: 동시성 테스트
  - 시나리오: 두 개의 다른 브라우저나 탭에서 동시에 같은 메뉴 항목의 가격을 다르게 수정하고 거의 동시에 저장합니다.

  - 예상 결과: 데이터베이스에 최종적으로 반영된 가격이 일관성을 유지하는지, 혹은 충돌이 발생했을 때 어떻게 처리되는지 확인합니다.

### 2-2. 반응형 및 UI/UX 테스트

- 시나리오 2-2-1: 모바일 및 다양한 화면 크기
  - 시나리오: 브라우저 창 크기를 줄여 모바일 환경처럼 만듭니다.

  - 예상 결과:
    - 모바일 화면에서는 우측의 모달/위젯 영역이 나타나지 않아야 합니다.
    - 전체적인 레이아웃(헤더, 메인, 푸터)이 깨지지 않고 정상적으로 표시되어야 합니다.
    - 가로 모드/세로 모드 전환 시에도 UI가 올바르게 렌더링되는지 확인합니다.

- 시나리오 2-2-2: 위젯 및 모달 동작
  - 시나리오:
    - 위젯을 열고 닫는 동작을 반복합니다.
    - 모달이 열려 있는 상태에서 다른 탭으로 전환을 시도합니다.
    - 모달 외부의 회색 배경(backdrop)을 클릭합니다.

  - 예상 결과:
    - 위젯의 열기/닫기 애니메이션이 부드럽게 동작해야 합니다.
    - 모달이 열려 있을 때는 탭 전환이 방지되어야 합니다.
    - 배경 클릭 시 모달이 정상적으로 닫혀야 합니다.

### 2-3. 실시간 기능 심화 테스트

- 시나리오 2-3-1: 다중 요청 및 주문 처리
  - 시나리오: 짧은 시간 안에 여러 테이블에서 동시에 주문과 요청을 발생시킵니다.

  - 예상 결과:
    - '주문' 탭의 '접수' 목록에 모든 주문이 누락 없이 순서대로 표시되어야 합니다.
    - 좌측 하단의 요청 알림이 쌓였다가 하나씩 처리될 때마다 정상적으로 사라지는지 확인합니다.

- 시나리오 2-3-2: 네트워크 연결 상태
  - 시나리오: 개발자 도구를 사용하여 네트워크를 오프라인으로 전환했다가 다시 온라인으로 복구합니다.

  - 예상 결과:
    - 화면 우측 상단의 '연결상태' 아이콘이 오프라인 시 빨간색, 온라인 복구 시 초록색으로 올바르게 변경되어야 합니다.
    - 연결이 복구되었을 때, 오프라인 동안 발생한 데이터 변경 사항(새 주문, 요청 등)이 화면에 올바르게 동기화되어야 합니다.
