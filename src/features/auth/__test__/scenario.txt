테스트 흐름 목록

1. `useAuthForm` 테스트:
    * 초기 상태: 폼의 초기 상태가 올바르게 설정되는지 확인.
    * 입력 변경: handleInputChange 호출 시 폼 상태가 업데이트되고, 해당 필드의 에러 메시지가 초기화되는지 확인. authStatus가 'error'일 때 'idle'로 변경되는지 확인.
    * 폼 초기화: params 변경 시 resetAllFormsAtom이 호출되어 폼 상태가 초기화되는지 확인.
    * 유효성 검사 성공: handleSubmit 호출 시 validationSchema.parse가 성공하고 onSubmit 콜백이 호출되는지 확인. setErrorForm이 빈 배열로 호출되는지 확인.
    * 유효성 검사 실패 (ZodError): handleSubmit 호출 시 validationSchema.parse가 실패하고 setErrorForm이 Zod 이슈와 함께 호출되는지 확인. authStatus가 'error'로 설정되는지 확인.
    * `onSubmit` 콜백 에러 (AuthError): handleSubmit 호출 시 onSubmit 콜백에서 AuthError가 발생했을 때 AuthErrorHandler가 호출되고 authStatus가 'error'로 설정되는지 확인.


2. `useAuthSession` 테스트:
    * 초기 세션 유효: 컴포넌트 마운트 시 supabase.auth.getSession이 유효한 세션을 반환할 때 isLogin이 true가 되고 authStatus가 'success'로 설정되는지 확인.
    * 초기 세션 무효: supabase.auth.getSession이 유효하지 않은 세션을 반환할 때 logoutAndClear가 호출되고 isLogin이 false가 되는지 확인.
    * JWT 검증 실패: verifyJWT가 null을 반환할 때 logoutAndClear가 호출되는지 확인.
    * `onAuthStateChange` - `SIGNED_OUT`: supabase.auth.onAuthStateChange에서 SIGNED_OUT 이벤트 발생 시 logoutAndClear 및 captchaRefresh가 호출되는지 확인.
    * `onAuthStateChange` - `SIGNED_IN`: SIGNED_IN 이벤트 발생 시 processSession이 호출되는지 확인.


3. `useDisabledState` 테스트:
    * authStatus와 captchaToken 값에 따라 disabled 상태가 올바르게 계산되는지 확인 (예: authStatus가 'idle'이고 captchaToken이 있을 때 false, 그 외에는 true).
    * authStatus 값이 올바르게 반환되는지 확인.


4. `useNavPage` 테스트:
    * disabled가 false일 때 handleNav 호출 시 resetAllFormsAtom이 호출되고 e.preventDefault()가 호출되지 않는지 확인.
    * disabled가 true일 때 handleNav 호출 시 e.preventDefault()가 호출되고 resetAllFormsAtom이 호출되지 않는지 확인.


5. `useSuccessRedirect` 테스트:
    * authStatus가 'success'일 때 REDIRECT_DELAY 시간 후 navigate가 PATHS.ROOT로 replace: true 옵션과 함께 호출되는지 확인.
    * authStatus가 'success'가 아닐 때 navigate가 호출되지 않는지 확인.


6. `AuthErrorHandler` 테스트:
    * ZodError 인스턴스 전달 시 setInputMessage가 올바른 issues 배열과 함께 호출되는지 확인.
    * AuthError 인스턴스 전달 시 error.code에 해당하는 authErrorHandler의 함수가 호출되는지 확인.
    * AuthError의 code가 captcha_failed일 때 captchaRefresh가 호출되는지 확인.
    * 알 수 없는 AuthError code에 대해 기본 오류 메시지가 showMessage를 통해 표시되는지 확인.
